Indicates that the SharpKit Build precompiler will generate code that will ensure that the property will never return a 'null' value. Instead, it will create a new instace from the specified type, and use it instead.

Syntax

C# 
public class AutoCreateAttribute : Attribute

Example

C#Copy Code
public class Contact 
{ 
 [AutoCreate] 
 public string Name {get;set;} 
  
 public static void Main(string[] args) 
 { 
    var contact = new Contact(); 
    Console.WriteLine("Name.length is {0}",contact.Name.Length); //This will print "Name.length is 0" to the console, and not throw an exception 
 } 
}

Remarks

The property type must expose the empty constructor with the proper accesibility level (usually public or internal) This attribute cannot be put on properties that hold a value type (structs, int, etc.)
